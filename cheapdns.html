cheapdns

IT'S SECURE BECAUSE IT'S HARD CODED(tm)

at this point, cheapdns is a weekend project implementing part of the dns standard as a minimal dns server. And minimal in this context means cheap. and by cheap, i mean "only answers plain, single-question A queries. and in answering these (and it answers all of them, no matter what domain name is specified), it gives one hard-coded IPv4 address using hard-coded values for authority and TTL. several of the strncpys also have hardcoded max lengths."

some amount of pains were taken to assure that though it may be cheap, and may have an almost undesirable amount of hardcoded behavior, this cheapness and behavioral defficiency would at least be consistent on unix-variant operating systems with different chip architectures. the code was compiled and tested on FreeBSD (i586), Solaris (sparc), MacOS X (ppc) and Linux (i586), and i don't believe any byte-order conversions have been neglected where they were necessary.

if its so cheap, then what is it good for?

Domain-authority dns serving an address for a virtual-hosting webserver. i.e. works well when queried by other nameservers. just don't point your host resolver directly at it.
